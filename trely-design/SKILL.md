## TL;DR — Что нужно помнить всегда

### Цвета (HEX из официального брендбука)
```
Deep Travel Green   #273C3F   Brand    — тёмный фон, dark cards, иногда текст
Explorer Gold       #C6A064   Brand    — главный акцент: цифры, иконки, активные пилюли, CTA
Soft Journey Beige  #F7F6F4            — основной фон светлых макетов, фон сайта
White               #FFFFFF            — карточки на светлом
Near-black          #0F1014            — основной текст на светлом
Warm tag/pill       #EDE4D1            — фон активной навигации, пилюли-теги
```
**Правило:** только зелёный + золото + кремовый. Никакого синего, красного, розового, фиолетового в UI. (Синие/фиолетовые акценты допустимы ТОЛЬКО внутри иллюстраций — на одежде персонажей.)

### Шрифт
**Inter Tight**, всегда. Веса: Light / Regular / Medium / SemiBold / Bold.
- Заголовки: Bold/SemiBold
- UI / подзаголовки: Medium
- Body: Regular
- line-height: tight (1.0–1.1)
- letter-spacing: −0.01em (плотно)

### Логотип
Изометрический куб = коробка + человек с поднятыми руками. 4 варианта: чёрный / золотой gradient / белый / на тёмно-зелёном. Всегда маленький, верхний-левый угол. Wordmark "Trely" — Inter Tight Medium с открытой буквой "y". **Готовые SVG/PNG лежат в `assets/logo/`** — используй их, не рисуй заново. Подробности и правила — `assets/logo/README.md`.

**Главное про gold-gradient версию:** градиент идёт **слева направо от near-black к gold**, буква "y" — самая светлая. Это парадная версия для обложек, не для UI-чипов.

## 🚨 КРИТИЧНОЕ ПРАВИЛО — Всегда используй реальные ассеты

**Никогда не оставляй макет без изображений.** В TRELY визуальные ассеты несут половину смысла бренда.

### Правило 1 — Иллюстрации
В любом макете (слайд, лендинг, обложка, креатив) **обязательно** вставляй фирменные painterly-иллюстрации:

| Где | Какой ассет ставить |
|---|---|
| Hero главной | `assets/illustrations/characters/02-mascot-main-green.png` или `06-travelclub-boy-bali.png` |
| Hero страницы Shipping | `assets/illustrations/characters/03-shipping-boy-luggage.png` |
| Hero страницы Car | `assets/illustrations/characters/04-car-duo-girls.png` |
| Hero страницы Travel Club / Services / Stay | `assets/illustrations/characters/06-travelclub-boy-bali.png` |
| Карточка сервиса с коробкой / иконка Shipping | `assets/illustrations/objects/01-box-green-stars.png` |
| Карточка сервиса с самолётом / Avia | `assets/illustrations/objects/02-plane-green-gold.png` |
| Карточка сервиса с машиной / Car | `assets/illustrations/objects/03-car-green-mini.png` |
| Карточка Travel / тропическое направление | `assets/illustrations/objects/04-palm-tropical.png` |

Полный мэппинг — `assets/illustrations/README.md`. Если нужного нет — генерь промтом из `prompts/illustrations.md` и **сохраняй результат в `assets/illustrations/`**.

❌ Не пиши `<img src="placeholder.png" />` или `<img src="path/to/character.png" />` — это бесполезно. Используй **реальный путь** к существующему файлу.

❌ Не используй эмодзи (🎁✈️🚗🌴) вместо иллюстраций.

### Правило 2 — Фото городов и направлений
Когда на странице маршрут / направление / город (Бали, Париж, Нью-Йорк и т.п.) — **обязательно** вставляй реальное фото:

**Самый быстрый способ — Unsplash Source URL:**
```html
<img src="https://source.unsplash.com/1200x800/?bali,temple" alt="Бали" />
<img src="https://source.unsplash.com/1200x800/?paris,eiffel" alt="Париж" />
```

Полный гайд с готовыми URL для 16+ городов — `references/imagery-sources.md`.

❌ Не оставляй пустые `<div>` с подписью "Бали" без фото.

### Правило 3 — Hero без иллюстрации = плохой макет
Любой hero обязан содержать хотя бы один из:
- Фирменную painterly иллюстрацию персонажа
- Объект-иллюстрацию (коробка/самолёт/машина/пальма)
- Реальное фото места (для маршрутных страниц)
- iPhone-моккап с интерфейсом (для секций про комьюнити)

Если ни одного нет — добавь.

### Правило 4 — Перед генерацией изображения проверь папку
**Шаг 1:** прежде чем писать промт для Nano Banana / Midjourney — открой `assets/illustrations/README.md` и проверь, есть ли уже нужный персонаж/объект. В 80% случаев он там.
**Шаг 2:** если нет — пиши промт из `prompts/illustrations.md`, генерь, **сохраняй в `assets/illustrations/`** под осмысленным именем для будущих макетов.

### Иллюстрации
Painterly watercolor/gouache, semi-realistic, тёплая текстура зерна. Палитра внутри иллюстрации: deep green + warm gold + denim blue + cream + чёрные акценты волос. **Никогда** flat-vector, 3D-render, фотореализм или иконки-стикеры в UI. Часто в **круглой виньетке** на светлом фоне с большой полупрозрачной аркой за объектом.

### Сетка
- Слайд: 1920×1080, паддинги 80–100px
- Сайт: max-width 1280–1440px, паддинги 24–32px на mobile, 64–96px на desktop
- Радиусы: 16px (UI) / 20–24px (карточки) / 28–32px (большие герой-карточки) / pill (для тегов)
- Тени: очень мягкие, тёплые — `0 8px 32px rgba(39,60,63,.06)` или `0 2px 12px rgba(0,0,0,.04)`

## Анатомия лендинга (trely.io / shipping.trely.io стандарт)

1. **Header** — sticky, белый или прозрачный на cream
   - Логотип-куб + "Trely" слева
   - Nav: 3–4 пункта (название сервиса / CTA-словарь / Войти)
   - Right: язык (RU/EN) + primary CTA `#0F1014` background

2. **Hero**
   - H1 огромный (60–80px), Inter Tight Bold
   - Sub: 20–22px, Regular, серый
   - 2 CTA: основная (dark BG, white text) + вторичная (outline)
   - Справа — иллюстрация маскота на полупрозрачной арке

3. **Сервисы Trely** (grid 3×2)
   - Карточки: white BG, radius 20-24px, иллюстрация-объект сверху, название Bold, описание Regular серым, статус-пилюля ("Скоро" / "Активно") в правом-верхнем углу
   - Active service — у карточки золотистый акцент или gold badge

4. **Как это работает** — 4 шага горизонтально
   - Номер шага в круге gold, заголовок Bold, описание Regular

5. **Почему нас выбирают** — 3-4 преимущества с золотой иконкой

6. **FAQ** — accordion, лёгкая разделительная линия

7. **Footer** — cream, ссылки серым, контакты, "Trely, 2026"

## Принцип "Move | Live | Connect"
Это формообразующая мантра бренда. Используется как **табы** или **разделитель** в hero под лого. Между словами — `|` (vertical bar) с тонким серым.

## Тоновый код контента (короткая выжимка)
- Заголовки **декларативные**, без вопросов: «Trely — твой мир путешествий в одном месте», не «Хочешь путешествовать?»
- Подзаголовки объясняют **что и зачем**, одно предложение
- Глаголы конкретные: "доставить", "найти", "забронировать", "вступить"
- Цифры — **золотом и крупно**, всегда с единицей измерения
- На каждом продуктовом блоке — короткая фраза-крючок («Зачем ждать и переплачивать? Найдите человека, который уже летит в нужный город.»)
- Статус-пилюли: «Скоро», «Запущено», «В планах»

Подробнее: `references/tone-of-voice.md`

## Workflow для каждого типа задачи

### "Сделай слайд X для презентации"
1. Загрузи `references/visual-dna.md` + `references/layouts.md`
2. Выбери макет A-F под тип контента
3. Если нужна иллюстрация — возьми готовый промт из `prompts/illustrations.md`
4. Собери HTML/CSS на базе `templates/slide-light.html` или `slide-dark.html`
5. Проверь по чек-листу в конце `visual-dna.md`

### "Сделай лендинг / секцию для сайта"
1. Загрузи `references/visual-dna.md` + `references/web-patterns.md`
2. Используй компоненты из `references/components.md`
3. Базовый шаблон — `templates/website-section.html`
4. Соблюдай иерархию: hero → сервисы → how-it-works → trust → FAQ → footer

### "Сгенерь иллюстрацию / обложку / креатив"
1. Открой `prompts/illustrations.md`
2. Подбери базовый промт (персонаж / объект / сцена)
3. Добавь обязательный suffix-блок с фирменными цветами и стилем
4. Если нужна композиция с текстом — добавь brand-frame (top-left логотип, cream BG)
